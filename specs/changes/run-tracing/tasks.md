- [x] Create `internal/version/version.go` with a `Version` string variable (default `"dev"`) and a function that falls back to `runtime/debug.ReadBuildInfo()` when the variable is unset per `Run Meta Event`
- [x] Create `internal/trace/events.go` with typed trace event structs for all 16 event types (`RunMetaEvent`, `StateChangeEvent`, `PhaseStartEvent`, `ExecutionContextEvent`, `ControllerActivityEvent`, `AgentInvokeEvent`, `AgentResultEvent`, `OutputChunkEvent`, `ReviewResultEvent`, `PhaseResultEvent`, `GitCommitEvent`, `GitPushEvent`, `PREvent`, `TauntEvent`, `SummaryEvent`, `RunEndEvent`) — all with `Type` and `Ts` JSON tags and the required fields per `Event Type Registry`
- [x] Create `internal/trace/writer.go` with `TraceWriter` struct: `New(path string, meta RunMetaEvent) (*TraceWriter, error)` creates the file with mode `0600`, writes `run_meta` as the first line, and starts the background goroutine. `Send(msg interface{})` does a non-blocking send to a buffered channel (cap 1024). `Close(runEnd RunEndEvent)` flushes the buffer, writes `run_end`, syncs the file, and waits for the goroutine to exit. Background goroutine reads from the channel, marshals to JSON, writes to a `bufio.Writer` (64KB). Error isolation: on I/O error, log to stderr and set a `failed` flag that drops all subsequent events per `Trace Writer Error Isolation` and `JSONL Trace Emission`
- [x] Add JSON marshaling from currently-existing pipeline event types to trace event types inside the background goroutine. The `internal/trace` package imports `internal/pipeline` for the type switch (`trace → pipeline` dependency direction). Map the 8 currently-existing pipeline event structs: `StateEvent` → `state_change`, `PhaseStartEvent` → `phase_start`, `OutputEvent` → `output_chunk`, `ExecutionContextEvent` → `execution_context`, `ControllerActivityEvent` → `controller_activity`, `TauntEvent` → `taunt`, `SummaryEvent` → `summary`, `PhaseResultEvent` → `phase_result` per `Trace Event Enrichment`
- [x] Write tests in `internal/trace/writer_test.go` for event serialization, file permissions (`0600`), error isolation (close file mid-run, verify no panic), channel overflow (fill channel, verify events are dropped not blocked), partial trace validity (every line is valid JSON), and `ReviewResultEvent` conditional `phase` field (present when `Step == "phase"`, omitted when `Step == "final"`) per `JSONL Line Schema` and `Trace Writer Error Isolation`
